# 贡献指南

感谢你考虑为 NotiMon 贡献代码！

## 提交 Issue

- **Bug 报告**：请描述运行环境（Android 版本、机型）和复现步骤
- **功能请求**：请清晰描述你期望的功能和使用场景

## 提交 PR

1. Fork 本仓库
2. 从 `main` 分支创建你的特性分支 (`git checkout -b feat/xxx`)
3. 提交改动 (`git commit -m 'feat: 添加 xxx 功能'`)
4. 推送到你的仓库 (`git push origin feat/xxx`)
5. 发起 Pull Request

## 代码风格

- 2 空格缩进
- 驼峰命名（camelCase）
- 函数名以动词开头
- 注释使用中文

## Commit 规范

| 类型 | 说明 |
|------|------|
| `feat:` | 新功能 |
| `fix:` | 修复 Bug |
| `refactor:` | 重构 |
| `docs:` | 文档更新 |
| `chore:` | 构建 / 工具链相关 |
